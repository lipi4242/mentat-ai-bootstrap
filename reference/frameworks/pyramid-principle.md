# Pyramid Principle

Barbara Minto's framework for top-down communication. Answer first, support second.

## What it is

State the answer (or recommendation) at the top. Below it, group 2–4 supporting arguments in MECE buckets. Under each argument, place 2–3 evidence items. The shape is a pyramid: one apex claim, a tier of grouped arguments, a base of evidence.

**SCQA hook** — use at the opening of a communication to set context:
- **Situation:** shared facts the audience already accepts
- **Complication:** what changed or went wrong
- **Question:** the question the complication raises
- **Answer:** your answer (= the apex of the pyramid)

SCQA is optional when the audience already knows the context. Skip it for short responses.

## When to apply

Any time you communicate a recommendation, status update, or synthesis — written or verbal. This is the agent's default output mode. Use it for: decision memos, analysis summaries, status reports, meeting prep.

## How to apply

1. **Write the apex claim in one sentence.** No hedging. "We should hire Ana." "The project is on track." "Cut Feature X."
2. **Identify 2–4 grouped supporting arguments.** Each group should be distinct (ME) and together cover all the support (CE). Name each group.
3. **Under each group, list 2–3 evidence items.** Facts, data, observations — not opinions.
4. **Check the structure:** does every evidence item support its group? Does every group support the apex?
5. **Write top-down.** Apex → groups → evidence. Never bury the lead.

## One worked example

**Question:** Should we hire this candidate?

> **Yes, hire Ana.**
>
> **Technical fit** — Strong Python and system design; passed the architecture round with no hints; prior work at scale (10M users).
>
> **Culture fit** — Explicit about async-first preferences; already uses similar tooling; asked good questions about documentation practices.
>
> **Growth trajectory** — Promoted twice in 3 years at previous company; clear learning pattern in take-home project (iterated on feedback, shipped v2 unprompted).
